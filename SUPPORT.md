# Support

## Questions and Usage Help

Use GitHub Discussions when it is enabled for:

- installation and build questions;
- help choosing commands or workflows;
- questions about `.snout` files;
- C ABI integration help;
- general design discussion.

If Discussions is not enabled, open a GitHub issue using the question option
and include enough context to reproduce the problem.

## Bug Reports

Use the bug report template for reproducible defects. Include:

- SnoutDB version or commit;
- Odin version;
- operating system and architecture;
- exact command;
- minimal input or fixture;
- actual and expected output.

Do not attach confidential production data. Reduce examples to the smallest
safe reproduction.

## Feature Requests

Describe the problem and workflow before proposing an implementation. Explain
why existing commands are insufficient and include representative input and
desired output where useful.

The project intentionally avoids networking, distributed systems, SQL
compatibility layers, and ORM concepts.

## Security

Do not use public issues for vulnerabilities. Follow [SECURITY.md](SECURITY.md).

## Scope

Support is provided on a best-effort basis. There are currently no guaranteed
response times or commercial support commitments.
